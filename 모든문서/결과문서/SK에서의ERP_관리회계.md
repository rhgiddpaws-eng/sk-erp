# SK ERP - 관리회계(CO) 영역 완전 분석

> 📅 작성일: 2026-02-24 | 대상: Web 연동 및 Web 분석 직무 지원자 참고용

---

## 1. 이게 어떤 부서인가? (쉬운 설명)

**관리회계(CO = Controlling)** 는 재무회계(FI)와 다릅니다. 쉽게 차이점을 말하면:

- 🔵 **재무회계(FI):** 외부(세무서, 투자자, 주주)에게 보고하는 회계 → "우리 회사 전체가 얼마 벌었어요"
- 🔴 **관리회계(CO):** 내부(경영진, 팀장들)에게 분석해주는 회계 → "A사업부가 B제품에서 얼마나 이익을 냈는지, 왜 비용이 늘었는지"

**관리회계 개발자(Web 분석 포지션)가 하는 일을 쉽게 말하면:**  
> "원가/이익/예산 데이터를 뽑아서, 경영진이 '이 사업이 돈이 되나요?'를 클릭 한 번에 알 수 있게 Web 분석 대시보드를 만들어주는 사람"

**SK그룹에서 특히 중요한 이유:** SK는 에너지, 반도체(SK하이닉스), 통신(SKT) 등 사업 부문이 매우 다양하기 때문에 **어느 사업부가 얼마나 수익을 내는지 원가 단위로 분석하는 능력**이 매우 중요합니다.

## 2. 실제 주요 업무 내용 (디테일)

| 업무 분류                       | 세부 내용                                                                                                     |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **원가 센터 분석**              | 부서별/제품별 원가를 분석하고 계획보다 실제가 얼마나 차이나는지(Variance) 보여주는 Web 리포트 개발            |
| **수익성 분석 화면 (CO-PA)**    | 제품/고객/사업부별 매출과 비용을 분석하여 순이익을 시각화하는 BI 대시보드 개발                                |
| **예산 관리 시스템**            | 예산을 입력하고, 실제 지출과 비교하여 초과/미달을 실시간으로 모니터링하는 Web 시스템 개발                     |
| **SAP BW/BI 데이터 파이프라인** | SAP ERP(FI/CO/MM 등)에서 나온 원시 데이터를 SAP BW(데이터 웨어하우스)로 이관하고 정제하는 ETL 파이프라인 개발 |
| **실시간 분석 대시보드**        | Power BI, SAP Analytics Cloud, Tableau 등과 연동하거나 직접 React/Next.js로 차트 대시보드를 만드는 업무       |
| **AI 기반 예측 분석**           | 과거 원가 데이터를 ML로 학습시켜 다음 분기 비용을 예측하는 모델 연동                                          |
| **통합 리포트 서버 구축**       | 여러 계열사의 CO 데이터를 취합하여 그룹 통합 관리 리포트 Web 시스템 구축                                      |

## 3. 핵심 기술 스택 상세

### 🔴 필수 (없으면 탈락 위험)
| 기술                                  | 설명                                                                                              |
| ------------------------------------- | ------------------------------------------------------------------------------------------------- |
| **SAP CO 모듈 이해**                  | 원가 센터(Cost Center), 내부 오더(Internal Order), 이익 센터(Profit Center), 수익성 분석(PA) 개념 |
| **SAP BW (Business Warehouse)**       | SAP 데이터 웨어하우스. 여기서 ERP 데이터를 꺼내서 분석용으로 가공함                               |
| **ETL 설계 (Extract-Transform-Load)** | 원본 DB에서 데이터를 뽑아서 가공하고 분석 DB에 넣는 파이프라인 설계                               |
| **SQL / HANA SQL**                    | SAP HANA는 In-Memory DB. HANA에 최적화된 SQL 쿼리 작성 능력                                       |

### 🟡 있으면 매우 유리
| 기술                          | 설명                                                                          |
| ----------------------------- | ----------------------------------------------------------------------------- |
| **SAP Analytics Cloud (SAC)** | SAP의 클라우드 기반 BI/Analytics 플랫폼. Excel처럼 쓰지만 Web으로 실시간 분석 |
| **Power BI / Tableau**        | Microsoft 또는 Tableau의 시각화 툴과 SAP ERP 데이터 연동 경험                 |
| **Python (Pandas, Numpy)**    | ERP 대용량 데이터를 파이썬으로 분석하고 리포트 생성                           |
| **React / Next.js 차트 개발** | Recharts, ECharts, D3.js 등을 활용한 커스텀 대시보드 개발                     |
| **ABAP**                      | SAP 전용 프로그래밍 언어. CO 모듈 커스터마이징에 활용                         |
| **Kafka / Spark**             | 대용량 재무 데이터 실시간 스트리밍 파이프라인                                 |

### 🟢 우대사항
- SAP BI(BW) 관련 자격증
- 원가회계 기초 개념 (고정비/변동비/한계이익 등)
- 데이터 엔지니어링 경험

---

## 4. 이 부서가 원하는 인재상

- **숫자를 이해하는 개발자.** 원가 데이터가 맞는지 회계팀 담당자와 검토하며 일할 수 있는 사람
- **대용량 데이터를 다루는 실력자.** 수억 건의 원가 전표 레코드를 빠르게 집계하고 보여줄 수 있는 DB/쿼리 최적화 실력
- **BI와 Web을 다 할 줄 아는 풀스택.** SAP BW에서 데이터를 가져다가 React 대시보드에 뿌려주는 End to End 개발 가능한 사람
- **AI 기반 분석 도입에 관심 있는 사람.** ML 모델로 비용 예측, 이상 감지 등을 ERP에 붙이는 트렌드 이해

## 5. 나의 이력서 기준 GAP 분석 (솔직하게)

| 항목                           | 보유 여부   | 설명                                                                     |
| ------------------------------ | ----------- | ------------------------------------------------------------------------ |
| SAP CO 모듈 운영/개발          | ❌ 없음      | 핵심 약점. 원가 센터/이익 센터 개념 사전 학습 필수                       |
| SAP BW / ETL 파이프라인        | ❌ 없음      | 학습 필요. 다만 Kafka/데이터 파이프라인 개념은 가지고 있음               |
| 대용량 데이터 집계/쿼리 최적화 | ✅ 강함      | 대규모 설문시스템(spring webflux, kafka) 및 게임 서버 DB 최적화 경험     |
| Web 대시보드/차트 개발         | ✅ 매우 강함 | 유치원 ERP Budget 모듈(예산/결산 대시보드), React/Next.js 화면 개발 완료 |
| Python 데이터 처리             | 🔶 기초 있음 | 이력서에 Python 명시, 고급 분석(Pandas) 역량 강조 필요                   |
| AI/ML 연동                     | ✅ 있음      | AI 생성형 관련 프로젝트(NCSoft AI 플랫폼 협업) 경험 있음                 |
| Kafka, 데이터 스트리밍         | ✅ 있음      | 엘리오소프트 대규모 설문시스템에서 Kafka 사용 경험                       |
| 관리회계 도메인 지식           | ❌ 약함      | 예산/결산 개념은 유치원 ERP로 이해, 원가회계는 별도 학습 필요            |

### 🎯 면접 보완 전략
1. **원가 관리의 핵심 용어 3개만 외우기:** 원가 센터(Cost Center), 이익 센터(Profit Center), 수익성 분석(Profitability Analysis)
2. 유치원 ERP에서 구현한 **예산(Budget) 대시보드, 결산 모듈(월별 비교 차트)**을 포트폴리오로 제시
3. "SAP CO 전문가와 함께 분석 요건을 정의하고, Web/데이터 파이프라인 구현을 담당할 수 있는 사람"으로 어필
4. AI NCSoft 협업 경험을 관리회계 예측 분석 맥락으로 연결
