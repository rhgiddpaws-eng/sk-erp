# SK ERP - 재무회계(FI) 영역 완전 분석

> 📅 작성일: 2026-02-24 | 대상: Web 연동 및 Web 분석 직무 지원자 참고용

---

## 1. 이게 어떤 부서인가? (쉬운 설명)

**재무회계(FI = Financial Accounting)** 는 쉽게 말해 **"회사가 돈을 어떻게 썼고, 얼마를 가지고 있는지 나라와 외부 투자자에게 보여주기 위한 회계"** 입니다.

- 세금계산서 처리, 외상매출/매입 관리, 은행 잔액 대조, 결산(월결/연결) 처리 등이 핵심 업무입니다.
- **SAP에서는 FI 모듈** 이라고 부릅니다.
- SK그룹에서는 여기에 추가로 AI/Web Dashboard 를 도입하여 실시간으로 재무현황을 한눈에 보게 하는 방향으로 가고 있습니다.

**ERP 재무회계 개발자(Web 연동/분석)가 하는 일을 아주 쉽게 말하면:**
> "SAP FI에 쌓인 복잡한 회계 데이터를, 경영진이 웹 브라우저(혹은 앱)에서 바로 클릭해 볼 수 있게 만들어주는 사람"

## 2. 실제 주요 업무 내용 (디테일)

| 업무 분류                   | 세부 내용                                                                                                        |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| **전표 처리 시스템 개발**   | 매입/매출 세금계산서를 국세청 e-세금계산서 시스템과 자동으로 연동하는 API 개발                                   |
| **은행 연동 (펌뱅킹)**      | 기업 은행 거래내역을 ERP와 자동 매칭하는 API 개발 (은행 FTP, API 연동)                                           |
| **전표 자동화**             | RPA 또는 OCR로 종이 영수증/전자세금계산서를 자동으로 ERP 전표로 입력하는 시스템                                  |
| **Web 대시보드**            | 매일/매주/매월 현금흐름(Cash Flow), 매출채권, 미지급금 등을 실시간으로 그래프로 보여주는 관리자 Web화면 개발     |
| **월 결산 자동화**          | 월말마다 사람이 수작업으로 하던 결산 작업을 프로그램이 순서대로 자동으로 실행하게 하는 배치(Batch) 프로그램 개발 |
| **SAP BTP (클라우드 확장)** | SAP가 제공하는 클라우드 플랫폼 위에서 새로운 웹앱 기능을 추가 개발                                               |
| **Fiori / UI5 화면 개발**   | SAP에서 제공하는 Web UI 프레임워크(Fiori/SAPUI5)를 이용하여 사용자 친화적인 업무 화면 개발                       |
| **ABAP 개발**               | SAP 전용 프로그래밍 언어(ABAP)로 리포트, 인터페이스, 업무 로직을 개발                                            |

## 3. 핵심 기술 스택 상세

### 🔴 필수 (없으면 탈락 위험)
| 기술                    | 설명                                                                               |
| ----------------------- | ---------------------------------------------------------------------------------- |
| **SAP FI 모듈 이해**    | 외상매출(AR), 외상매입(AP), 총계정원장(GL), 자산(AA), 자금(TR) 의 개념과 설정 방법 |
| **ABAP**                | SAP 전용 서버 개발 언어. Java나 C#과 비슷하지만 SAP 안에서만 돌아가는 언어         |
| **SQL / Oracle / HANA** | SAP HANA DB는 SAP S/4HANA의 핵심 DB. SQL 쿼리 최적화 능력 필수                     |
| **REST API 개발**       | SAP와 외부 시스템(국세청, 은행, 그룹웨어 등) 연동 인터페이스 개발                  |

### 🟡 있으면 매우 유리 (10점 추가)
| 기술                                       | 설명                                                              |
| ------------------------------------------ | ----------------------------------------------------------------- |
| **SAP Fiori / SAPUI5**                     | SAP가 제공하는 반응형 웹 UI 프레임워크 (React.js와 개념이 유사함) |
| **SAP BTP (Business Technology Platform)** | SAP 클라우드 플랫폼. 여기서 새로운 확장 앱을 개발                 |
| **Java Spring Boot**                       | ERP 연동용 미들웨어 서버 개발                                     |
| **국세청 API 연동 경험**                   | 홈택스 전자신고, e세금계산서 API                                  |

### 🟢 우대사항
- SAP FI 또는 FICO 관련 자격증
- 더존 iCUBE, 영림원 K-system 등 국산 ERP 경험
- 회계(부기) 기초 지식

## 4. 이 부서가 원하는 인재상

- **회계는 이미 아는 사람(회계팀)과 함께 일하며**, 그 사람들의 업무 방식을 **웹/IT 시스템으로 구현해주는 개발자**
- ERP 데이터가 많아서 느린 리포트를 **빠르게 최적화**할 줄 아는 사람 (쿼리 튜닝, 캐싱)
- 레거시 ABAP 코드를 이해하고 최신 Web(Fiori, React) 화면으로 현대화할 수 있는 사람
- 회계 부서 담당자와 직접 인터뷰하며 요구사항을 정의하고 개발로 연결하는 **분석/기획력**

## 5. 나의 이력서 기준 GAP 분석 (솔직하게)

| 항목                               | 보유 여부   | 설명                                                                |
| ---------------------------------- | ----------- | ------------------------------------------------------------------- |
| SAP FI 모듈 운영/개발 경험         | ❌ 없음      | 가장 핵심 약점. 면접 전 SAP FI 기초개념은 꼭 숙지                   |
| ABAP 개발 경험                     | ❌ 없음      | SAP 전용 언어라 단기 학습 필요                                      |
| REST API / 연동 개발               | ✅ 충분      | 게임 서버, 호텔 시스템, 유치원 ERP 경험으로 충분히 커버             |
| Java Spring Boot 웹 서버           | ✅ 충분      | 게임잡 이력서에 spring boot 경험 명시됨                             |
| 대용량 트랜잭션 처리               | ✅ 매우 강함 | 게임 서버 대규모 동접 처리 경험 직결                                |
| SQL / DB 최적화                    | ✅ 충분      | MSSQL, MySQL, Oracle 경험 있음                                      |
| Web 대시보드 개발 (React, Next.js) | ✅ 매우 강함 | 유치원 ERP Web (Next.js 16)에서 직접 구현                           |
| 회계 도메인 기초 지식              | 🔶 약함      | 유치원 ERP에서 전표/결산 모듈 개발 경험 있으나 기업재무와 차이 있음 |

### 🎯 면접 보완 전략
1. SAP FI의 핵심 용어(AR, AP, GL, 전표, 결산)를 검색해서 개념만 파악 (2~3일)
2. "SAP FI 전문가와 협업하여 Web 연동 및 데이터 파이프라인 개발"을 담당하겠다는 포지션으로 이야기
3. 유치원 ERP에서 **현금출납, 전표 모듈을 Next.js + Prisma로 개발했다**는 것을 최대한 강조
