# SK ERP - 구매 영역(MM/SCM) 완전 분석

> 📅 작성일: 2026-02-24 | 대상: Web 연동 및 Web 분석 직무 지원자 참고용

---

## 1. 이게 어떤 부서인가? (쉬운 설명)

**구매(Procurement / MM = Materials Management)** 는 쉽게 말해 **"회사가 필요한 물건이나 서비스를 어디서, 얼마에, 어떻게 사는지 관리하는 프로세스"** 입니다.

SK그룹처럼 큰 회사는 매년 수백조 원어치 이상의 부품, 원료, 서비스를 외부 협력업체로부터 구매합니다. 이 구매 과정이 디지털화되지 않으면 비효율이 엄청납니다.

**구매 영역 개발자(Web 연동/분석)가 하는 일을 쉽게 말하면:**  
> "협력업체가 Web 포털에 접속해서 견적을 넣고, 발주를 확인하고, 납품을 확인하고, 대금을 청구하는 B2B 공급업체 포털 시스템을 만들어주는 사람"  
> + "SCM(공급망) 데이터를 분석해서 어디서 납기 지연이 발생하고 있는지, 어떤 협력사가 문제인지 대시보드로 보여주는 사람"

---

## 2. 실제 주요 업무 내용 (디테일)

| 업무 분류                                  | 세부 내용                                                                                             |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------- |
| **구매 요청/발주 Web 시스템**              | 내부 직원이 웹에서 구매 요청을 올리면, 자동으로 발주서가 생성되고 협력업체에 발송되는 워크플로우 개발 |
| **협력업체(Vendor) Web 포털**              | 협력업체가 웹으로 로그인해서 발주 확인, 납품 통보, 대금 청구를 할 수 있는 B2B 포털 개발               |
| **조달청 / 국세청 API 연동**               | 나라장터(G2B), e세금계산서, ETC. 공공 API와 ERP를 자동 연계하는 시스템                                |
| **SCM 데이터 파이프라인 및 분석 대시보드** | 납기 준수율, 협력사 평가, 구매 단가 트렌드 등을 실시간 Web 대시보드로 분석                            |
| **AI/ML 수요 예측**                        | 과거 구매 이력을 분석하여 다음 달 필요한 자재 물량을 예측, 자동 발주 추천 시스템 개발                 |
| **BOM(자재 명세서) 관리 Web 화면**         | 제품을 만들 때 필요한 모든 부품 목록(Bill of Materials)을 입력/조회/수정하는 화면 개발                |
| **SAP MM / Ariba 연동**                    | SAP의 구매 모듈(MM)이나 클라우드 구매 솔루션(Ariba)과 자체 개발 Web 시스템 연동                       |
| **RPA(로봇 자동화) 구매 업무 자동화**      | 이메일로 오는 견적서를 OCR로 읽어 ERP에 자동 입력, 또는 반복 발주 자동화                              |

## 3. 핵심 기술 스택 상세

### 🔴 필수 (없으면 탈락 위험)
| 기술                              | 설명                                                                                 |
| --------------------------------- | ------------------------------------------------------------------------------------ |
| **SAP MM 모듈 이해**              | 구매 오더(PO), 입고(GR), 송장(IR/LIV), 자재 마스터 등의 구매 프로세스 이해           |
| **REST API / Web Service**        | ERP와 협력업체 포털, 외부 시스템(나라장터, 관세청 등) 간 데이터 연동 인터페이스 개발 |
| **Java Spring Boot 또는 Node.js** | 구매 포털 백엔드 서버 개발                                                           |
| **Web Frontend (React/Vue)**      | 협력업체 포털 화면, 구매 대시보드 개발                                               |

### 🟡 있으면 매우 유리
| 기술                                  | 설명                                                                 |
| ------------------------------------- | -------------------------------------------------------------------- |
| **SAP Ariba**                         | SAP의 클라우드 기반 구매/조달 플랫폼. 대기업 협력업체 관리에 많이 씀 |
| **SCM 솔루션 경험**                   | JDA, O9 Solutions 등 공급망 계획 솔루션 연동 경험                    |
| **AI/ML (수요 예측 모델)**            | Python(scikit-learn, Prophet)을 이용한 수요 예측 모델 구축           |
| **EDI (Electronic Data Interchange)** | 표준 전자 문서 교환 포멧으로 협력업체와 데이터 자동 교환             |
| **Kafka / RabbitMQ**                  | 대량 구매 이벤트(발주, 입고, 검수)를 실시간으로 처리하는 메시지 큐   |
| **BOM 데이터 모델링**                 | 복잡하게 중첩된 자재 구조(제품-반제품-원자재) 를 DB로 설계하는 능력  |

### 🟢 우대사항
- 산업공학/물류 전공
- CPIM (생산재고관리사) 자격증
- 제조업이나 물류 도메인 경험

## 4. 이 부서가 원하는 인재상

- **B2B 포털을 만들어본 경험이 있는 사람.** 협력업체처럼 '외부인'이 쓰는 Web 시스템을 만들어본 경험
- **복잡한 비즈니스 워크플로우를 시스템으로 구현하는 능력.** 구매 요청 → 결재 → 발주 → 입고 → 검수 → 대금지급의 단계별 흐름을 프로그램으로 구현
- **API 연동 전문가.** 국세청, 관세청, 나라장터 등 외부 공공 API와 SAP 내부를 연결하는 브리지 개발 능력
- **데이터를 보고 인사이트를 주는 대시보드 개발자.** 구매 비용 절감 기회를 찾아주는 분석 화면

## 5. 나의 이력서 기준 GAP 분석 (솔직하게)

| 항목                         | 보유 여부   | 설명                                                         |
| ---------------------------- | ----------- | ------------------------------------------------------------ |
| SAP MM 모듈 경험             | ❌ 없음      | 구매 PO/GR/IR 프로세스 기초 학습 필요                        |
| B2B 외부 API 연동            | ✅ 강함      | 공진원(공공기관) 시스템 연동, 아이돌 팬플랫폼 외부 API 경험  |
| Web 포털(외부 사용자용) 개발 | ✅ 강함      | 협력업체 포털과 유사한 외부 사용자 Web 시스템 개발 경험 다수 |
| React/Next.js 프론트엔드     | ✅ 매우 강함 | 유치원 ERP, 호텔시스템 등 다수 React 기반 Web 개발 경험      |
| Java Spring Boot 백엔드      | ✅ 있음      | 대규모 설문시스템에서 Spring Boot + Kafka 사용 경험          |
| Node.js / NestJS 백엔드      | ✅ 있음      | 이력서에 nestjs 경험 명시                                    |
| AI/ML 수요 예측              | 🔶 간접 경험 | NCSoft AI 플랫폼 협업 경험 있으나 구매 예측 特化는 아님      |
| SCM 구매 도메인 지식         | ❌ 없음      | 구매 프로세스(PO, GR, 검수) 개념 사전 학습 필요              |
| EDI / 공공 API 연동          | 🔶 있음      | 공공기관 시스템 연동 경험 있으나 EDI 포멧 경험은 불명확      |

### 🎯 면접 보완 전략
1. **구매 프로세스 5단계를 외워두기:** "구매 요청(PR) → 발주(PO) → 입고(GR) → 송장검증(IV) → 대금지급(Payment)"
2. 아이돌 팬플랫폼의 **대규모 외부 사용자 API 연동**과 협력업체 포털을 유사 경험으로 연결
3. 공진원(공공기관) 시스템 연동 경험을 **나라장터/공공 API 연동의 전 단계 경험**으로 어필
4. 호텔 PMS 리뉴얼 프로젝트를 **"업무 프로세스가 복잡한 B2B 시스템을 리모델링한 경험"** 으로 강조
