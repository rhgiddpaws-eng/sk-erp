# SK ERP - 장비/설비 영역(PM/EAM/CMMS) 완전 분석

> 📅 작성일: 2026-02-24 | 대상: Web 연동 및 Web 분석 직무 지원자 참고용

---

## 1. 이게 어떤 부서인가? (쉬운 설명)

**장비/설비(PM = Plant Maintenance / EAM = Enterprise Asset Management)** 는 쉽게 말해 **"공장이나 사업장에 있는 기계, 설비, 장비가 잘 돌아가고 있는지 관리하고, 고장 나기 전에 미리 점검하는 업무"** 를 시스템화한 것입니다.

- 예: SK하이닉스 반도체 공장의 수천 대 생산 설비가 각각 잘 가동 중인지?
- 예: SK에너지(주유소/정유공장) 설비들의 점검 이력, 부품 교체 일정은?
- 예: SKT(SK텔레콤) 기지국 장비의 고장 이력과 유지보수 일정은?

**장비/설비 영역 개발자(Web 연동/분석)가 하는 일을 쉽게 말하면:**  
> "수천 개의 설비에서 올라오는 센서 데이터/가동률/고장 데이터를 Web 대시보드에서 실시간으로 보여주고, AI로 '이 기계가 2주 후에 고장 날 것 같습니다'라고 예측해주는 시스템을 만드는 사람"

이 분야는 최근 **IoT + AI + 디지털트윈** 이 접목되면서 가장 핫한 ERP 파트 중 하나입니다.

## 2. 실제 주요 업무 내용 (디테일)

| 업무 분류                               | 세부 내용                                                                                     |
| --------------------------------------- | --------------------------------------------------------------------------------------------- |
| **설비 정보 관리 Web 시스템**           | 설비 마스터 데이터(제작사, 모델, 설치일, 위치, 담당자 등)를 입력/관리하는 Web 화면 개발       |
| **예방 정비(PM) 스케줄러**              | 정기적인 점검/교체 일정을 자동으로 생성하고 담당 기사에게 모바일 알림을 보내는 시스템         |
| **설비 이상 감지 대시보드**             | 센서(온도, 진동, 압력, 전류 등)에서 올라오는 실시간 데이터를 Web 대시보드에 시각화            |
| **AI 예지보전(Predictive Maintenance)** | 머신러닝으로 센서 데이터 패턴을 학습하여 고장 발생 시점을 미리 예측하는 AI 모델 연동          |
| **디지털트윈 Web 뷰어**                 | 실제 공장 설비 배치를 3D/2D Web으로 렌더링하고 실시간 상태를 색깔/지시등으로 표시             |
| **작업 지시(Work Order) 시스템**        | 고장이 접수되면 자동으로 작업 지시서가 생성되고, 기사가 모바일로 점검 완료 처리를 하는 시스템 |
| **IoT 데이터 파이프라인 개발**          | 공장 PLC/SCADA/IoT 장비에서 오는 데이터를 수집하여 ERP/DW로 전달하는 파이프라인               |
| **유지보수 비용 분석 리포트**           | 설비별, 기간별 유지보수 비용과 가용성(Availability) 분석 Web 리포트                           |

## 3. 핵심 기술 스택 상세

### 🔴 필수 (없으면 탈락 위험)
| 기술                    | 설명                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------- |
| **SAP PM 모듈 이해**    | 기능적 위치(Functional Location), 장비(Equipment), 작업 지시(Work Order), 알림(Notification) 개념 |
| **IoT/MQTT/OPC-UA**     | 공장 센서 장비에서 데이터를 수집하는 프로토콜. MQTT는 IoT의 TCP/IP 같은 존재                      |
| **REST API / Web 개발** | IoT 데이터를 ERP에 연동하고, Web 대시보드에 보여주는 인터페이스 개발                              |
| **시계열 데이터 처리**  | 센서 데이터는 시간 순서로 끊임없이 쌓이는 데이터. InfluxDB, TimescaleDB 등 시계열 DB 이해         |

| 기술                               | 설명                                                                           |
| ---------------------------------- | ------------------------------------------------------------------------------ |
| **디지털트윈 기술**                | Unity3D / Babylon.js / Three.js 등으로 공장 설비를 3D Web 시각화               |
| **Python (scikit-learn, LSTM)**    | 설비 센서 데이터로 고장 예측 ML 모델 개발                                      |
| **Kafka / MQTT Broker**            | 수천 개의 센서에서 동시에 올라오는 데이터를 처리하는 메시지 スポーツ파이프라인 |
| **Apache Spark / Flink**           | 대용량 설비 로그 데이터 실시간 처리                                            |
| **GIS (지도 기반 설비 위치 관리)** | 공장/부지 지도 위에 설비 위치를 표시하고 상태를 관리하는 Web GIS 시스템        |
| **SAP CMMS / EAM**                 | Infor EAM, IBM Maximo 등 전문 설비관리 솔루션 연동 경험                        |
| **React / Three.js / WebGL**       | Web 브라우저에서 3D 공장 뷰어 개발                                             |

- 제조업/플랜트 현장 경험
- 디지털트윈 구축 경험

## 4. 이 부서가 원하는 인재상

- **공장/설비란 어떻게 돌아가는지 이해하는 개발자.** 현장 엔지니어(기계담당자)와 직접 소통하며 요구사항을 정리할 수 있는 사람
- **실시간 데이터를 다루는 능력.** 설비 데이터는 끊임없이 들어오므로, 스트리밍 데이터 처리와 시각화 모두 가능한 사람
- **3D/2D 기반 Web 시각화 가능한 개발자.** 디지털트윈 수요가 높아지면서 Unity나 Three.js 경험이 큰 플러스
- **AI 예지보전 개발 경험자.** ML/DL로 이상 감지 모델을 만들어 본 경험이 최우선 관심사
- **IoT와 웹을 동시에 아는 풀스택.** 센서 → 수집 서버 → DB → Web 대시보드까지 전 과정을 이해하는 사람

## 5. 나의 이력서 기준 GAP 분석 (솔직하게)

| 항목                         | 보유 여부         | 설명                                                                |
| ---------------------------- | ----------------- | ------------------------------------------------------------------- |
| SAP PM 모듈 경험             | ❌ 없음            | FI/CO보다는 격차가 적음. 기본 개념 학습으로 보완 가능               |
| **디지털트윈 개발 (LG CNS)** | ✅ **초강력 무기** | 배터리 공장 내부 설계를 Unity로 시뮬레이션한 (업계 최고 연관 경험!) |
| **Unity 3D 엔지니어링**      | ✅ 매우 강함       | 게임 + LG CNS 디지털트윈 모두에서 Unity 숙련                        |
| IoT/MQTT 데이터 처리         | ❌ 없음            | 그러나 게임서버의 실시간 통신(소켓, protobuf) 경험이 직결됨         |
| 실시간 데이터 스트리밍       | ✅ 있음            | 게임 서버 대규모 실시간 이벤트 처리 경험 (C++ IOCP, Kafka)          |
| 3D/2D Web 시각화             | 🔶 간접 경험       | Three.js, cocos creator, PixiJs는 이력서에 기재됨                   |
| Python ML / 예지보전         | 🔶 기초            | NCSoft AI 협업 경험, Python 사용 기재                               |
| 공장/설비 도메인             | ✅ **있음**        | LG CNS 배터리 공장 디지털트윈 프로젝트로 1년 경험                   |
| Web 대시보드 (실시간 차트)   | ✅ 매우 강함       | 유치원 ERP, 호텔 시스템 등 다수 Web 대시보드 개발 경험              |

### 🎯 면접 보완 전략 (이 분야가 최고 확률!)
1. **"LG CNS 배터리 공장 디지털트윈"을 첫 번째 카드로 꺼내세요!** 이 경험 하나로 설비 도메인의 이해도를 증명 가능
2. 게임 서버의 **C++ IOCP 실시간 소켓 통신** 경험을 **"IoT 센서 데이터 실시간 수집과 원리가 동일"** 하다고 연결
3. Unity를 활용한 3D 뷰어 개발 경험을 **디지털트윈 Web 시각화**로 직결
4. 아래 포인트를 면접에서 반드시 언급:
   - "MQTT 프로토콜은 생소하지만, 게임 서버에서 protobuf 기반 실시간 이벤트 처리와 원리가 동일합니다. 빠르게 적응할 수 있습니다."
   - "공장 설비의 실시간 모니터링 대시보드는 게임 운영 서버 모니터링 대시보드와 구조가 유사합니다. 이미 그 종류의 시스템을 만들어 왔습니다."
