## 구현할 내용에 대한 조사 결과

실제 은행 계좌 연동 방식
업로드하신 아이큐브 매뉴얼에 나온 "빠른계좌조회" 또는 "스피드조회"는 실제로 은행과 연동하는 방식입니다. 말씀하신 것처럼 단순히 DB에 저장만 하는 게 아닙니다.

1. 빠른계좌조회 서비스 (스피드조회)
방식: 사용자가 각 은행 인터넷뱅킹에서 직접 "빠른계좌조회" 서비스를 신청하고, 해당 계좌를 등록
인증: 은행에 로그인 ID/PW를 저장하거나, 은행에서 발급한 조회 전용 API 키를 사용
특징:
인증서 없이도 계좌 조회 가능
주로 세무사, 회계사가 고객 계좌 조회용으로 사용
스크래핑 방식을 사용 (은행 로그인 후 HTML 파싱)

2. 오픈뱅킹 API (2019년 도입)
방식: 금융결제원이 제공하는 표준화된 API
인증: OAuth 2.0 방식 (사용자가 직접 계좌 연동 동의)
특징:
실제 은행 시스템과 직접 연동
핀테크이용번호(가상계좌번호) 발급
잔액조회, 거래내역조회, 이체 등 가능
스크래핑보다 안전하고 빠름


## 아이큐브 시스템이 사용하는 방식
매뉴얼을 보면 "빠른계좌조회 서비스" = 스크래핑 방식을 사용하는 것으로 보입니다:
1. 유치원이 각 은행 인터넷뱅킹에서 "빠른계좌조회" 신청
2. 아이큐브 시스템이 저장된 은행 로그인 정보로 자동 로그인
3. 계좌 거래내역 HTML 데이터를 크롤링(스크래핑)
4. 데이터 파싱 후 아이큐브 DB에 저장
스크래핑 vs 오픈뱅킹 API 차이
구분스크래핑오픈뱅킹 API인증 방식ID/PW 저장OAuth (사용자 동의)데이터 수집HTML 파싱표준 API 응답안정성은행 사이트 변경시 오류안정적속도느림 (로그인 필요)빠름보안상대적으로 취약안전

결론
아이큐브는 실제 은행과 연동하되, "스크래핑" 방식을 사용합니다. 정부기관 사이트에 대리 입력하는 것이 아니라, 은행 시스템에 직접 접근하여 데이터를 가져옵니다.
최신 핀테크 서비스들은 "오픈뱅킹 API"를 사용하는 추세지만, 아이큐브 같은 기존 ERP 시스템들은 여전히 스크래핑 방식을 활용하고 있습니다.
